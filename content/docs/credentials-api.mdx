Provider credentials let Hyperstruck call the model provider behind an agent. They are required for agent execution and for manual learning storage on that agent.

<Callout type="warning" title="Why this exists">
  Hyperstruck resolves credentials by provider. The agent's `model_provider` must match an active credential's `provider`, or runs cannot start and manual learning store requests for that agent fail.
</Callout>

<Callout type="info" title="Recommended default">
  For most teams, a `tenant_default` credential is the fastest way to get the first agent running. Use an `agent_override` only when one agent should use a different provider key than the tenant default.
</Callout>

## What provider credentials do

- They tell Hyperstruck which provider key to use at runtime.
- They are selected by `provider`, not by arbitrary label.
- They must match the agent's `model_provider`.

## Upload a tenant-default credential

<TypeTable
  type={{
    provider: {
      description: 'Model provider this credential applies to.',

    },
    display_name: {
      description: 'Human-readable label for the credential.',
      type: 'string',
      required: true,
    },
    binding_type: {
      description: 'Credential binding mode.',
      type: '"tenant_default"',
      required: true,
    },
    secret: {
      description: 'Provider API key or secret.',
      type: 'string',
      required: true,
    },
    is_active: {
      description: 'Whether the credential is active.',
      type: 'boolean',
      default: 'true',
      required: false,
    },
    metadata: {
      description: 'Optional provider metadata, including `metadata.base_url`.',
      type: 'object',
      default: '{}',
      required: false,
    },
  }}
/>

Do not include `agent_id` when `binding_type` is `tenant_default`.

```bash
curl -X POST \
  -H "Authorization: Bearer <API_KEY>" \
  -H "Content-Type: application/json" \
  "https://api.core.hyperstruck.com/credentials/providers" \
  -d '{
    "provider": "anthropic",
    "display_name": "Default Anthropic Key",
    "binding_type": "tenant_default",
    "secret": "<ANTHROPIC_API_KEY>"
  }'
```


## Common next knobs

`metadata.base_url` is optional.

If `metadata.base_url` is omitted:

- `openai` stores `https://api.openai.com/v1`
- `ollama` stores `https://ollama.com/api`
- `anthropic` and `groq` do not get a default `base_url`

Legacy `metadata.endpoint` is accepted, but public docs should prefer `metadata.base_url`.

## Runtime credential resolution

Hyperstruck resolves credentials in this order:

1. It checks the agent's `model_provider`.
2. A matching active `agent_override` credential for that agent wins first.
3. If no matching `agent_override` exists, Hyperstruck falls back to the tenant's matching active `tenant_default` credential.
4. If neither exists, agent runs cannot start and manual learning store requests for that agent fail.

This is why Quickstart begins with a `tenant_default` credential. It is the fastest path to a working first run.

## How this affects runs and manual learning

- Agent execution uses the credential that matches the agent's `model_provider`.
- Manual learning storage for an agent follows the same provider-matching requirement.
- Teams only need `agent_override` when one agent should use a different provider key than the tenant default.
