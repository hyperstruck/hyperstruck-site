This page walks through the smallest useful Hyperstruck workflow for a brand-new user: create an agent, run a goal, optionally add a manual learning, and reuse the same capability from developer tooling.

<Callout type="idea" title="What this shows">
  The goal is not a full production integration. It is a working first path for someone who has an API key, no existing agent, and no prior learnings.
</Callout>

<Callout type="info" title="Automatic and manual learning">
  Reasoning runs can automatically extract useful learnings from execution. The manual learning step below is optional and is best used for explicit tuning, curation, imported lessons, or operator-injected guidance.
</Callout>

## Minimal flow

<Steps>
  <Step>

### Create an agent

Create an agent first so you have an `AGENT_ID` to use in later runs.

```bash
curl -X POST \
  -H "Authorization: Bearer <API_KEY>" \
  -H "Content-Type: application/json" \
  "https://api.core.hyperstruck.com/agents" \
  -d '{
    "name": "Operations Assistant",
    "model_provider": "anthropic",
    "model_name": "claude-sonnet-4-5",
    "reasoning_profile": "balanced",
    "memory_profile": "default",
    "knowledge_scope": "agent",
    "core_config": {
      "instructions": "You are a helpful operations assistant."
    }
  }'
```

  </Step>
  <Step>

### Dispatch a goal

Use the returned `AGENT_ID` to start a reasoning run.

```bash
curl -X POST \
  -H "Authorization: Bearer <API_KEY>" \
  -H "Content-Type: application/json" \
  "https://api.core.hyperstruck.com/agents/<AGENT_ID>/goals" \
  -d '{
    "goal": "Map out a migration plan with milestones and risks",
    "context": "Current stack, constraints, and known blockers go here"
  }'
```

Hyperstruck can automatically extract useful learnings from what happens during the run when meaningful patterns emerge.

  </Step>
  <Step>

### Optionally store a manual learning

Use the Learnings API when you want direct editorial control over what the system should carry forward.

```bash
curl -X POST \
  -H "Authorization: Bearer <API_KEY>" \
  -H "Content-Type: application/json" \
  "https://api.core.hyperstruck.com/agents/<AGENT_ID>/learnings" \
  -d '{
    "content": "Always validate the migration dependency graph before scheduling milestones.",
    "learning_type": "best_practice",
    "confidence": 0.7
  }'
```

  </Step>
  <Step>

### Reuse the same capabilities from developer tooling

Once the APIs are in place, you can expose the same reasoning and manual learning controls through Claude Code or Cursor skills.

```text
/hyper-reasoning Map out a migration plan with milestones and risks
/hyper-learning search migration dependency graph
/hyper-learning store
```

  </Step>
</Steps>

## How to think about this flow

- [Reasoning](/docs/reasoning) improves decision quality during the run.
- [Learning](/docs/learning) improves future decisions by carrying useful lessons forward.
- [Agents API](/docs/agents-api) is the orchestration surface for creating agents, dispatching goals, and monitoring runs.
- [Learnings API](/docs/learnings-api) is the manual control surface for injecting, curating, retrieving, and reinforcing learnings.
- [Claude Skills](/docs/claude-skills) makes the same APIs easy to use where developers already work.
