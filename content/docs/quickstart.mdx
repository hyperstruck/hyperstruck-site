This is the fastest first-success path: create an agent, dispatch a goal, and optionally store a manual learning.

<Callout type="info" title="Provider credentials are optional">
  Hyperstruck can run agents with a platform-provided model credential. Upload your own provider credential only when you need a tenant default or agent-specific override.
</Callout>

<Callout type="info" title="Keep the first call small">
  Start with the minimum required fields. Full API reference lives in the OpenAPI docs. If you omit optional fields here, Hyperstruck uses sane defaults.
</Callout>

## Minimal flow

<Steps>
  <Step>

### 1. Create an agent

Create an agent with a `name` and `core_config.instructions`. If you omit provider and model fields, Hyperstruck intelligently uses the best-suited model by default.

```bash
curl -X POST \
  -H "Authorization: Bearer <API_KEY>" \
  -H "Content-Type: application/json" \
  "https://api.core.hyperstruck.com/agents" \
  -d '{
    "name": "Sales Execution Assistant",
    "description": "Helps sales reps plan follow-ups, next steps, and account strategy.",
    "core_config": {
      "instructions": "You are a helpful sales execution assistant."
    }
  }'
```

  </Step>
  <Step>

### 2. Dispatch a goal

Use the returned `AGENT_ID` to start a run.

```bash
curl -X POST \
  -H "Authorization: Bearer <API_KEY>" \
  -H "Content-Type: application/json" \
  "https://api.core.hyperstruck.com/agents/<AGENT_ID>/goals" \
  -d '{
    "goal": "Create a follow-up sales execution plan for a call with client XYZ"
  }'
```

  </Step>
  <Step>

### 3. Optionally store a manual learning

Reasoning runs can already extract learnings automatically. Use manual storage when you want direct editorial control.

```bash
curl -X POST \
  -H "Authorization: Bearer <API_KEY>" \
  -H "Content-Type: application/json" \
  "https://api.core.hyperstruck.com/agents/<AGENT_ID>/learnings" \
  -d '{
    "content": "At client XYZ, directors and below are usually influencers, not final decision-makers, so map the buying committee before assigning next steps.",
    "learning_type": "pitfall"
  }'
```

  </Step>
  <Step>

### 4. Reuse the same capabilities from developer tooling

Once the API flow works, you can expose the same capabilities in developer tooling.

```text
/hyper-reasoning Create a follow-up sales execution plan for a call with client XYZ
/hyper-learning search client XYZ decision makers
/hyper-learning store
```

  </Step>
</Steps>

## Common next knobs

- Agent creation defaults: `status="active"`, automatic model selection, `reasoning_profile="full"`, `memory_profile="default"`, `knowledge_scope="agent"`
- Goal dispatch optional fields: `context`, `session_id`, `worker_profile`, `metadata`
- Manual learning optional fields: `confidence`, `source_goal`, `applicable_goals`, `applicable_tools`, `privacy`

## When to upload credentials

You usually do not need provider credentials for the first run because Hyperstruck provides a platform default. Upload credentials only when you want model billing, provider selection, or endpoint control under your own account.

Credential resolution is:

1. A matching active `agent_override` credential for the agent wins first.
2. If none exists, Hyperstruck uses a matching active `tenant_default`.
3. If neither exists, Hyperstruck uses the platform default credential and intelligently selects the best-suited model.

Use [Credentials API](/docs/credentials-api) when you want to bring your own provider account.
