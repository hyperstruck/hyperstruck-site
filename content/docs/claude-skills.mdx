Claude Skills let teams try Hyperstruck directly inside Claude Code and Cursor without building a heavy integration first. They expose reasoning and learning flows through lightweight commands that fit into existing developer workflows.

Claude skills expose both hosted reasoning and manual learning controls inside developer workflows. They are not a separate product surface. They are a workflow-friendly layer on top of the same APIs.

<Callout type="success" title="Why teams start here">
  Claude Skills are a low-friction way to try Hyperstruck in real developer workflows before committing to a deeper SDK or product integration.
</Callout>

## How it fits into the workflow

A lightweight integration path for Claude Code and Cursor that exposes Hyperstruck reasoning and learning capabilities through skills.

## Why it lands quickly

Teams can start using Hyperstruck where work is already happening, which lowers adoption friction and makes the platform easier to try in real workflows.

## What teams can do first

- use hosted reasoning from an IDE workflow
- search and store learnings during real work
- bring Hyperstruck into developer tooling quickly

## Suggested install copy

Install the public Hyperstruck skills from `https://github.com/hyperstruck/public-integrations` and use them to invoke hosted reasoning and learning flows directly from Claude Code or Cursor.

## Example commands

```text
/hyper-reasoning Map out a migration plan with milestones and risks
/hyper-learning search retry strategy
/hyper-learning store
```
