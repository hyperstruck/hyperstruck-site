Hyperstruck can already extract learnings automatically from reasoning runs. The Learnings API is the lighter manual control surface for teams that want to inject, curate, search, and reinforce specific lessons.

<Callout type="info" title="Start small">
  Start with the minimum required fields. Full API reference lives in the OpenAPI docs.
</Callout>

## Store a learning

<TypeTable
  type={{
    content: {
      description: 'The learning text to store.',
      type: 'string',
      required: true,
    },
    learning_type: {
      description: 'The type of learning being stored.',
      type: '"tool_usage" | "approach" | "pitfall" | "prerequisite" | "coordination_pattern" | "agent_capability" | "conflict_insight" | "debate_outcome" | "tool_behaviour" | "outcome_pattern" | "coverage"',
      required: true,
    },
    confidence: {
      description: 'Confidence score for the learning.',
      type: 'number',
      default: '0.5',
      required: false,
    },
    source_goal: {
      description: 'Goal text the learning came from.',
      type: 'string',
      default: '""',
      required: false,
    },
    applicable_goals: {
      description: 'Goals this learning is especially relevant to.',
      type: 'string[]',
      default: '[]',
      required: false,
    },
    applicable_tools: {
      description: 'Tools this learning is especially relevant to.',
      type: 'string[]',
      default: '[]',
      required: false,
    },
    privacy: {
      description: 'Privacy setting for the learning.',
      type: '"shareable" | "agent_specific" | "sensitive"',
      default: '"shareable"',
      required: false,
    },
    instances: {
      description: 'Optional structured evidence examples for this learning.',
      type: 'InstanceEvidenceInput[]',
      default: '[]',
      required: false,
    },
    'instances[].entity_values': {
      description: 'Non-empty map of input entity names to string values.',
      type: 'Record<string, string>',
      required: true,
    },
    'instances[].outcome': {
      description: 'Non-empty map of outcome names to string values.',
      type: 'Record<string, string>',
      required: true,
    },
    'instances[].source_context': {
      description: 'Optional source label for this evidence instance.',
      type: 'string',
      default: '"api"',
      required: false,
    },
  }}
/>

```bash
curl -X POST \
  -H "Authorization: Bearer <API_KEY>" \
  -H "Content-Type: application/json" \
  "https://api.core.hyperstruck.com/agents/<AGENT_ID>/learnings" \
  -d '{
    "content": "qualify_lead returns cold for cybersecurity companies",
    "learning_type": "outcome_pattern",
    "applicable_tools": ["qualify_lead"],
    "instances": [
      {
        "entity_values": {
          "company": "CrowdShield",
          "industry": "cybersecurity"
        },
        "outcome": {
          "tier": "cold",
          "score": "0.12"
        },
        "source_context": "api"
      }
    ]
  }'
```

This endpoint returns `202 Accepted` and stores the learning asynchronously. Background processing handles validation and indexing, so it may take a moment before the learning appears in search results.

`instances` is optional and backward-compatible. Use it for concrete input-to-outcome examples, tool behavior observations, test cases, compacted external-run evidence, and other structured evidence. Do not store raw logs, secrets, PII, or internal hostnames.

Each evidence instance must include non-empty `entity_values` and `outcome` maps with string keys and string values. If `source_context` is omitted, the platform API defaults it to `api`. API-sourced evidence is content-addressed with `InstanceEvidence.make_id(None, entity_values=..., outcome=...)`, so duplicate entity/outcome examples collapse into one instance.

Required scope: `agents:write`

Valid `learning_type` values:

- `tool_usage`
- `approach`
- `pitfall`
- `prerequisite`
- `coordination_pattern`
- `agent_capability`
- `conflict_insight`
- `debate_outcome`
- `tool_behaviour`
- `outcome_pattern`
- `coverage`

## Search learnings

The only required query param is `q`.

```bash
curl -H "Authorization: Bearer <API_KEY>" \
  "https://api.core.hyperstruck.com/agents/<AGENT_ID>/learnings/search?q=migration+dependency+graph"
```

<TypeTable
  type={{
    q: {
      description: 'Search text.',

    },
    limit: {

`org` scope searches shared learnings across agents and is available on eligible plans.

Required scope: `agents:read`

Search responses may include compact returned evidence on each learning:

```json
          },
          "source_context": "api",
          "created_at": "2026-05-17T02:01:00Z"
        }

    }
  ]
}
```

## Get a learning

No body or query params are required.

```bash
curl -H "Authorization: Bearer <API_KEY>" \
  "https://api.core.hyperstruck.com/agents/<AGENT_ID>/learnings/<LEARNING_ID>"
```

Required scope: `agents:read`

Read responses may include the full evidence instances currently attached to the learning:

```json
{
  "learning_id": "learning-1",
  "content": "qualify_lead returns cold for cybersecurity companies",
  "learning_type": "outcome_pattern",
  "applicable_tools": ["qualify_lead"],
  "instances": [
    {
      "id": "instance-1",
      "entity_values": {
        "company": "CrowdShield",
        "industry": "cybersecurity"
      },
      "outcome": {
        "tier": "cold",
        "score": "0.12"
      },
      "source_context": "api",
      "created_at": "2026-05-17T02:01:00Z"
    }
  ]
}
```

## Reinforce a learning

<TypeTable
  type={{
    is_helpful: {
      description: 'Whether the retrieved learning was helpful.',
      type: 'boolean',
      required: true,
    },
  }}
/>

```bash
curl -X POST \
  -H "Authorization: Bearer <API_KEY>" \
  -H "Content-Type: application/json" \
  "https://api.core.hyperstruck.com/agents/<AGENT_ID>/learnings/<LEARNING_ID>/reinforce" \
  -d '{
    "is_helpful": true
  }'
```

Required scope: `agents:write`

## When to use manual learning

- Use automatic learning first for low-friction compounding improvement.
- Use manual learning when you want direct editorial control over a lesson.
- Use reinforcement when you want to strengthen or down-rank a retrieved learning based on whether it helped.
