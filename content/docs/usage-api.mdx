Use the Usage API to see aggregate usage and recent run history for your own tenant.

<Callout type="info" title="Lead with the default tenant view">
  Start with the no-query-params version first. Add filters only when you need them.
</Callout>

## Own-tenant endpoints

### `GET /usage/summary`

No query params are required.

```bash
curl -H "Authorization: Bearer <API_KEY>" \
  "https://api.core.hyperstruck.com/usage/summary"
```

## Common next knobs

- Use `window` to switch from the default `last_7_days` view.
- Use `limit` and `cursor` when paging through run history.
- Use `as_of` when you want a report anchored to a specific point in time.

## Advanced admin endpoints

These public endpoints are for operator-style cross-tenant reporting and should come after the default own-tenant flow:

### `GET /usage/tenants/{tenant_id}/summary`

```bash
curl -H "Authorization: Bearer <API_KEY>" \
  "https://api.core.hyperstruck.com/usage/tenants/<TENANT_ID>/summary"
```

### `GET /usage/tenants/{tenant_id}/runs`

```bash
curl -H "Authorization: Bearer <API_KEY>" \
  "https://api.core.hyperstruck.com/usage/tenants/<TENANT_ID>/runs"
```

These admin endpoints require `usage:admin:read`.
